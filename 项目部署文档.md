### 二、一键部署失败后的手动配置（修改 yii.bat 的 PHP 路径）

当一键部署脚本失败（通常提示找不到 PHP、或命令无法执行）时，可以按以下步骤手动完成配置。核心思路是把 `yii.bat` 中的 PHP 路径改为本机真实的 `php.exe` 路径，然后逐步执行初始化命令。

#### 1. 确认本机 PHP 路径

以 XAMPP 为例，默认路径为：

```
G:\xampp\php\php.exe
```

如果你的 XAMPP 安装路径不同，请替换为实际路径。

#### 2. 修改 yii.bat 中的 PHP 路径

打开项目根目录下的 `yii.bat`，找到类似以下内容：

```
@echo off
php "%~dp0yii" %*
```

将 `php` 替换为完整路径，例如：

```
@echo off
G:\xampp\php\php.exe "%~dp0yii" %*
```

保存文件后关闭。

#### 3. 逐步执行手动配置命令

在项目根目录打开命令行（PowerShell 或 CMD），先运行 `init.bat`，选择开发环境（Development）并确认。

```
init.bat
```

随后执行 Yii 初始化入口（会使用你在 `yii.bat` 中配置的 PHP 路径）：

```
yii.bat
```

先修改数据库配置（必须步骤）：

打开并编辑：

```
..\common\config\main-local.php
```

根据你的本机数据库信息，修改 `db` 配置中的 `dsn`、`username`、`password` 等字段并保存。

然后执行数据库迁移：

```
./yii migrate
```

如有需要，执行前后端资源初始化（按项目实际情况）：

```
./yii asset
```



#### 4. 验证

再次访问项目主页或部署页面，确认页面正常加载：

```
http://localhost/../frontend
```

如仍失败，请检查 `config` 中的数据库连接信息是否正确，以及 `runtime`、`web/assets` 是否有写入权限。
